Membangun Sistem POS dari Nol: Apa yang Saya Pelajari

Fathahaz Nur Ramdhani 6 menit baca Node.js · Express · MySQL

Salah satu proyek yang paling banyak memberi saya pelajaran teknis adalah AvePOS — sebuah sistem point-of-sale untuk kebutuhan kedai kopi dan retail, dibangun menggunakan vanilla JavaScript di sisi front-end serta Node.js, Express, dan MySQL di sisi back-end. Tulisan ini merangkum proses, tantangan, dan pelajaran yang saya ambil selama mengembangkannya.

Latar Belakang

AvePOS dibangun untuk menjawab kebutuhan operasional kedai kopi yang saya kelola sendiri, Hazzcoffee — mulai dari pencatatan transaksi harian, manajemen produk, hingga laporan penjualan. Daripada mengandalkan aplikasi POS generik, saya memilih membangun sistem sendiri agar alur kerja benar-benar sesuai dengan kebutuhan operasional di lapangan.

Tantangan Teknis

Beberapa tantangan yang muncul selama development antara lain:

  • Manajemen state keranjang (cart) — memastikan penambahan, pengurangan, dan penghapusan item pada keranjang tetap konsisten tanpa framework state management, murni menggunakan vanilla JavaScript.
  • Konsistensi format tanggal — perbedaan format tanggal antara input form, penyimpanan di database, dan tampilan laporan sempat menyebabkan beberapa bug yang cukup sulit dilacak.
  • Upload gambar produk — implementasi upload gambar produk menggunakan multer di sisi Express, termasuk validasi tipe file dan pengelolaan path penyimpanan di server.
  • Struktur basis data — merancang skema MySQL yang cukup fleksibel untuk menampung produk, transaksi, dan relasi antar tabel tanpa membuat query menjadi terlalu kompleks.

Contoh Pendekatan: Update Keranjang

Salah satu bagian yang paling sering di-refactor adalah logika pembaruan keranjang. Pendekatan yang akhirnya saya pakai kurang lebih seperti berikut (disederhanakan untuk ilustrasi):

function updateCartItem(cart, productId, qty) {
    const item = cart.find(i => i.productId === productId);
    if (!item) return cart;

    item.qty = Math.max(0, qty);
    if (item.qty === 0) {
        return cart.filter(i => i.productId !== productId);
    }
    return cart;
}

Pola sederhana ini membantu memastikan satu source of truth untuk state keranjang, sehingga bug terkait jumlah item yang tidak sinkron antar komponen tampilan bisa diminimalkan.

Pelajaran yang Saya Ambil

Ada beberapa hal yang menurut saya cukup penting dari proses ini:

  • Menulis README dan dokumentasi skema database sejak awal sangat membantu ketika proyek harus di-maintain kembali setelah beberapa waktu.
  • Vanilla JavaScript tetap relevan untuk proyek berskala kecil-menengah, selama struktur kode dan pemisahan tanggung jawab dijaga dengan disiplin.
  • Menangani edge case seperti format tanggal lebih baik diselesaikan di satu layer (misalnya di back-end) daripada ditangani terpisah di banyak tempat.

Kesimpulan

Membangun AvePOS dari nol mengajarkan saya bahwa kompleksitas sebenarnya bukan pada teknologi yang dipakai, tetapi pada bagaimana menjaga konsistensi data dan alur kerja tetap sederhana untuk pengguna akhir. Proyek ini menjadi salah satu fondasi penting sebelum saya mengerjakan sistem yang lebih besar seperti HAZZSTORE.ID.
