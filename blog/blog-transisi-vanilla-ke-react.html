Dari Vanilla JS ke React: Transisi yang Tidak Semudah Kelihatannya

Fathahaz Nur Ramdhani 5 menit baca React · Vite · JavaScript

Banyak yang bilang kalau sudah paham Vanilla JavaScript, belajar React akan terasa jauh lebih mudah. Pernyataan itu tidak sepenuhnya salah, tapi juga tidak sepenuhnya benar. Perubahan terbesar bukanlah pada syntax, melainkan pada cara kita memikirkan bagaimana sebuah UI dibangun.

Paradigma Imperatif vs Deklaratif

Saat menggunakan Vanilla JS, saya terbiasa memberikan perintah langkah demi langkah (imperatif). Ambil elemen A, ubah warnanya, buat elemen B, lalu masukkan ke dalam A. Fokusnya adalah bagaimana cara mengubah DOM.

Ketika beralih ke React, saya dipaksa berpikir secara deklaratif. Saya hanya perlu memberi tahu React seperti apa UI yang saya inginkan berdasarkan *state* tertentu, dan React yang akan mengurus pembaruan DOM-nya. Transisi mindset ini adalah tantangan terbesar di minggu-minggu awal saya.

Komponen: Memecah UI Menjadi Puzzle

Di proyek-proyek awal Vanilla JS saya, satu file HTML bisa berisi ratusan baris kode. CSS dan logika JS-nya juga terpisah di file raksasa lainnya. React mengajarkan saya tentang Component-Based Architecture.

  • Reusability: Membuat tombol *custom* sekali, lalu memanggilnya berkali-kali di berbagai halaman hanya dengan <Button />.
  • Maintainability: Jika ada bug di tampilan keranjang, saya tahu persis saya harus mengecek Cart.jsx tanpa harus menggulir ribuan baris kode.

Jebakan *State* dan *Lifecycle*

Meninggalkan manipulasi DOM langsung dan beralih menggunakan useState dan useEffect terasa seperti belajar berjalan lagi. Dulu, jika saya ingin mengambil data API saat halaman dimuat, saya cukup menulis fetch(). Di React, saya harus memahami *lifecycle* komponen agar tidak terjebak dalam *infinite loop re-rendering* (percayalah, laptop saya sempat *freeze* beberapa kali karena ini).

Kesimpulan

Mempelajari Vanilla JS secara mendalam (lewat kelas Dicoding, misalnya) terbukti sangat membantu. Saya tidak bingung lagi dengan ES6 Modules, Destructuring, atau Higher-Order Functions seperti map dan filter yang dipakai setiap saat di React.

Transisi ke React memang memaksa otak untuk berpikir dengan cara yang baru. Tapi setelah melewati fase frustrasinya, development terasa jauh lebih cepat, terstruktur, dan tentu saja... lebih menyenangkan!
